## 一、研究背景与核心问题

### 1.1 态势感知与意图推理的战略意义

战场态势感知与意图推理是智能化作战决策系统的核心能力，直接决定了智能体在复杂对抗环境中的生存能力和作战效能。在庙算军棋平台的分队级陆战推演场景中，态势感知与意图推理面临以下核心挑战：

**信息不对称挑战**。庙算平台采用不完全信息博弈规则，博弈双方仅能获取己方传感器感知范围内的战场信息，对感知范围外的敌方单位位置、状态、意图等关键情报处于未知状态。这种信息不对称性构成了态势感知和意图推断的根本性挑战，要求智能体具备在信息缺失条件下进行有效推理的能力。

**动态对抗挑战**。战场态势随时间持续演变，敌方单位可能采取隐蔽机动、佯攻欺骗、战术切换等多种对抗策略。态势感知系统需要具备实时更新能力，能够根据新获取的侦察信息快速调整态势认知；意图推断系统需要具备预测能力，能够在敌方才取行动之前预判其下一步意图。

**复杂环境挑战**。六角格战场空间包含多种地形要素（山地、平原、水域、道路、建筑物等），不同地形对观察视野、机动能力、通信传输等产生差异化影响。态势感知算法需要综合考虑地形遮蔽、掩蔽条件、高差影响等多种环境因素，实现对战场空间的准确建模。

### 1.2 核心研究目标

本项目围绕态势感知与意图推理两大核心能力，制定了以下具体研究目标：

**态势感知层面**：构建六角格战场空间的高效表示与计算框架，实现对可见敌目标的准确识别和定位；开发多模态信息融合算法，整合雷达侦察、光学侦察、电子侦察等多种感知手段的信息；建立威胁评估模型，对各类目标进行威胁等级排序；生成可视化态势图，为决策层提供直观的态势表征。

**意图推断层面**：建立敌方行为序列的建模与特征提取方法，实现对行为模式的标准化表示；构建基于贝叶斯推断的意图识别引擎，支持在不完全信息条件下的意图推断；开发多假设追踪框架，同时维护多个可能的意图假设及其置信度；实现意图预测功能，在时间维度上延伸对敌方行动的预判。

## 二、态势感知算法研究进展

### 2.1 态势感知算法体系架构

态势感知算法体系采用"感知-认知-表达"三层架构设计，自底向上依次为数据感知层、态势认知层和态势表征层。

**态势感知算法架构图**：

![态势感知算法架构](mermaid://sa_architecture)

**数据感知层**负责从平台SDK获取原始观察数据，包括己方单位状态列表、可见敌单位列表、地形信息、通信情报等。该层的核心任务是将SDK数据格式转换为内部标准数据结构，为上层算法提供统一的输入接口。

**态势认知层**是态势感知的核心算法模块，包含以下功能组件：

- **敌我识别组件**：基于单位类型、阵营标识、行为特征等信息，判定观察到的目标是友方、敌方还是中立单位；
- **位置精化组件**：对观察到的目标位置进行精度评估和不确定性量化，生成带置信度的位置估计；
- **状态估计组件**：推断敌我单位的当前状态，包括活动状态（移动/静止）、战斗状态（展开/收起）、载具状态（上车/下车）等；
- **威胁评估组件**：综合考虑目标类型、数量、位置、朝向等因素，计算各目标的威胁等级。

**态势表征层**负责将态势认知结果整合为标准化的态势表征格式，支持以下输出形式：网格化态势图（以六角格为基本单元的态势栅格）、对象化态势表（以作战单位为对象的态势清单）、区域化威胁云（表示威胁等级分布的热力图）。

### 2.2 敌我识别算法

敌我识别是态势感知的首要任务，直接影响后续所有态势分析的可信度。现阶段实现的敌我识别算法综合运用多种识别判据，形成多级筛选的识别流程。

**识别判据体系**：

| 判据类型 | 判据内容 | 可信度权重 | 适用条件 |
|---------|---------|-----------|---------|
| 阵营标识 | SDK直接提供的faction字段 | 100% | 始终可用 |
| 单位外观 | 平台提供的单位类型和外观特征 | 80% | 可见单位 |
| 行为模式 | 单位的行为特征（移动模式、射击习惯） | 60% | 有历史数据 |
| 位置关系 | 与已知友军/敌军的相对位置 | 40% | 有参照目标 |
| 通信特征 | 通信情报中涉及的单位标识 | 70% | 有通信截获 |

**识别算法流程**。首先检查SDK数据中是否包含可靠的阵营标识，如有则直接返回识别结果；如无则综合运用其他判据进行推断。当多种判据存在冲突时，采用加权投票机制确定最终识别结果，同时输出识别置信度。

**算法实现代码位置**：`agent.py`中通过`observation.get("operators")`获取单位列表，结合`self.faction`进行阵营匹配。

### 2.3 通视分析算法

通视分析是态势感知的核心技术之一，用于判断两个位置之间是否存在视觉遮挡关系，直接影响可见性判定和侦察规划。

**通视分析算法核心**。在六角格战场空间中，通视分析需要考虑以下因素：

- **地形遮蔽**：高地、建筑物等对视线产生遮挡的地形要素；
- **高度影响**：观察者与目标的高度差影响遮挡判定结果；
- **距离衰减**：观察距离超过一定阈值后观察能力下降；
- **单位遮蔽**：己方或敌方单位可能临时遮挡观察视线。

**多模通视判定**。算法`can_see(pos1, pos2, mode)`支持四种观察模式：

```python
def can_see(pos1, pos2, mode=0):
    # mode=0: 地面观察地面
    # mode=1: 空中观察空中  
    # mode=2: 空中观察地面
    # mode=3: 地面观察空中
```

每种模式对应不同的遮挡判定规则和距离限制，通过`mode`参数在预计算的通视矩阵中查询判定结果。

**通视矩阵预计算**。为提高实时判定效率，系统预计算了四层通视矩阵，分别对应四种观察模式。通视矩阵`see[mode][row1, col1, row2, col2]`存储位置对(row1,col1)到(row2,col2)的通视判定结果，支持O(1)时间复杂度的实时查询。

**观察遮蔽判定算法**。综合通视分析结果和遮蔽因素，`check_2bop_see(bop_a, bop_b, mod)`函数实现完整的观察遮蔽判定流程：

```python
def check_2bop_see(bop_a, bop_b, mod=0):
    # 1. 获取观察距离约束
    max_distance = get_observation_distance(bop_a, bop_b)
    
    # 2. 检查距离是否满足
    hex_distance = get_distance(pos_a, pos_b)
    if hex_distance > max_distance:
        return False
    
    # 3. 执行通视检查
    if not can_see(pos_a, pos_b, mode):
        return False
    
    # 4. 应用遮蔽修正
    distance_threshold = apply_obscuring_effects(bop_a, bop_b, max_distance)
    
    # 5. 返回综合判定结果
    return hex_distance <= distance_threshold
```

**遮蔽效应修正规则**：

| 目标状态 | 修正系数 | 修正说明 |
|---------|---------|---------|
| 掩蔽地形内（森林、建筑） | 0.5 | 有效观察距离减半 |
| 隐蔽状态（步兵隐藏） | 0.25 | 隐蔽单位更难被发现 |
| 高位观察低位 | 1.5 | 高位观察者具有优势 |
| 低位观察高位 | 0.75 | 低位观察受限 |

### 2.4 距离计算与空间分析

六角格距离计算是态势感知的基础算法，为目标定位、威胁评估、路径规划等提供空间度量支撑。

**六角格曼哈顿距离算法**。在六角格坐标系中，采用轴向坐标表示，两点间的曼哈顿距离计算公式为：

```
q =col - (row - (row & 1)) // 2
r = row
s = -q - r
distance = (|q1 - q2| + |r1 - r2| + |s1 - s2|) // 2
```

该距离度量满足三角不等式性质，适用于六角格战场空间中的各种空间度量需求。

**环形区域分析**。`get_grid_distance(center, r0, r1)`算法生成以中心点为原点、指定距离范围内的环形区域坐标集合，支持战术场景中的火力覆盖分析、侦察范围评估等应用。

```python
def get_grid_distance(center, r0, r1):
    """获取距离中心点 [r0, r1] 范围内的所有六角格坐标"""
    grids = set()
    row, col = divmod(center, 100)
    
    # 遍历边界框内的所有位置
    for row_x in range(max(0, row-r1), min(max_row, row+r1+1)):
        for col_x in range(max(0, col-r1), min(max_col, col+r1+1)):
            pos = row_x * 100 + col_x
            dist = get_distance(center, pos)
            if r0 <= dist <= r1:
                grids.add(pos)
    
    return grids
```

**PM观察范围计算**。针对特定侦察需求，实现了PM（观察所）观察范围的精确计算：

```python
def get_PM_scope(grid):
    """计算PM在指定格子的观察范围（排除遮蔽区域）"""
    # 基础观察范围：距离2以内的所有格子
    base_scope = get_grid_distance(grid, 0, 2)
    # 排除遮蔽区域内的格子
    cover_grids = get_grid_in_cover(get_grid_distance(grid, 2, 2))
    return base_scope - cover_grids
```

### 2.5 威胁评估模型

威胁评估是态势感知的核心输出之一，为决策层提供敌目标的威胁等级排序，支持资源优化配置和火力分配决策。

**威胁因素建模**。威胁评估综合考虑以下因素：

| 威胁因素 | 权重 | 说明 |
|---------|-----|------|
| 目标类型 | 0.25 | 装甲单位>步兵>后勤单位 |
| 距离因素 | 0.20 | 距离越近威胁越大 |
| 火力能力 | 0.20 | 视武器类型和射程 |
| 态势朝向 | 0.15 | 面向我方的威胁更大 |
| 活动状态 | 0.10 | 机动中更危险 |
| 协同关系 | 0.10 | 与友军协同增加威胁 |

**威胁等级计算公式**：

```
ThreatScore = Σ(Wi × Fi)
其中：
Wi = 第i个威胁因素的权重
Fi = 第i个威胁因素的归一化分数（0-1）
```

**威胁等级划分**：

| 威胁分数区间 | 威胁等级 | 颜色标识 | 建议响应 |
|------------|---------|---------|---------|
| 0.0-0.3 | 低威胁 | 绿色 | 常规监视 |
| 0.3-0.6 | 中威胁 | 黄色 | 重点关注 |
| 0.6-0.8 | 高威胁 | 橙色 | 优先应对 |
| 0.8-1.0 | 极高威胁 | 红色 | 立即处置 |

## 三、意图推理算法研究进展

### 3.1 意图推理算法框架

意图推理是智能决策系统的核心认知能力，目标是从观测到的敌方行为中推断其作战意图，为主动对抗提供决策依据。

**意图推理框架架构**：

![意图推理框架](mermaid://ir_framework)

意图推理框架包含以下核心组件：

- **行为采集模块**：收集敌方单位的历史行为序列；
- **特征提取模块**：从原始行为数据中提取高层语义特征；
- **意图建模模块**：建立意图假设空间和先验分布；
- **推断引擎模块**：基于贝叶斯推断计算各意图假设的后验概率；
- **置信评估模块**：评估推断结果的可信程度。

### 3.2 行为序列建模

行为序列建模是意图推断的基础，将时间序列的原始行为数据转换为可推理的语义表示。

**行为事件定义**。系统定义了以下行为事件类型用于行为序列建模：

| 事件类型 | 事件参数 | 语义含义 |
|---------|---------|---------|
| 移动事件 | 起点、终点、速度、移动模式 | 位置变化行为 |
| 射击事件 | 目标位置、武器类型、结果 | 火力使用行为 |
| 状态变更 | 原状态、目标状态 | 状态转换行为 |
| 编组变更 | 变化类型、参与单位 | 队形调整行为 |
| 进入/退出工事 | 工事位置 | 隐蔽行为 |

**行为特征提取**。从行为序列中提取以下高层特征：

**运动学特征**：

```
- 平均速度向量：反映整体机动方向
- 速度方差：反映机动稳定性
- 轨迹曲率：反映机动复杂度
- 方向变化频率：反映意图不确定性
```

**战术特征**：

```
- 与我方目标的距离变化趋势：接近/远离
- 与关键地域的位置关系：威胁指向
- 与友军的协同程度：配合意图
- 战术机动模式：包围/突击/侦察等
```

**行为序列表示**。采用变长向量表示行为序列，固定维度表示各时刻的行为特征向量，序列长度由历史窗口大小决定。当前实现采用时间窗口大小为10个决策周期。

### 3.3 意图假设空间定义

意图假设空间定义了智能体需要推断的所有可能意图类别，是意图推断的语义基础。

**基础意图类型**。系统定义了以下四类基础作战意图：

| 意图类型 | 意图代号 | 战术含义 |
|---------|---------|---------|
| 进攻意图 | ATTACK | 向我方目标推进并意图打击 |
| 防御意图 | DEFENSE | 占据有利地形准备抗击 |
| 侦察意图 | RECON | 隐蔽接近获取我方情报 |
| 撤退意图 | RETREAT | 脱离接触保存实力 |

**意图组合表示**。实际作战中，敌方可能同时存在多种意图（如"边打边撤"），系统采用组合向量表示意图分布：

```
意图向量 = [进攻倾向, 防御倾向, 侦察倾向, 撤退倾向]
各分量取值范围 [0, 1]，归一化和为1
```

**意图层级结构**。系统建立了三层意图层级结构：

```
战略意图层：决定战争走向的宏观意图（进攻/防御/相持）
战术意图层：决定战场态势的中观意图（突破/包围/佯攻）
行动意图层：决定具体动作的微观意图（移动方向/射击目标）
```

高层意图通过约束低层意图的可能性空间来影响决策，形成自上而下的意图传递机制。

### 3.4 贝叶斯推断引擎

贝叶斯推断是意图推理的核心算法，通过整合先验知识和观测证据，计算意图假设的后验概率分布。

**贝叶斯推断公式**：

```
P(意图 | 行为) ∝ P(行为 | 意图) × P(意图)

其中：
P(意图) = 意图先验概率，反映任务背景和态势的先验知识
P(行为 | 意图) = 似然函数，反映给定意图下产生特定行为的概率
P(意图 | 行为) = 意图后验概率，推断的最终目标
```

**先验概率建模**。意图先验概率P(意图)根据以下因素动态调整：

```python
def compute_intention_prior(scenario_info, current_stage):
    """根据场景信息和当前阶段计算意图先验"""
    prior = {
        'ATTACK': 0.3,
        'DEFENSE': 0.3,
        'RECON': 0.3,
        'RETREAT': 0.1
    }
    
    # 根据当前阶段调整先验
    if current_stage == 'EARLY':
        prior['RECON'] += 0.15  # 早期侦察意图可能性增加
        prior['ATTACK'] -= 0.10
    elif current_stage == 'LATE':
        prior['ATTACK'] += 0.15  # 后期进攻意图可能性增加
        prior['RECON'] -= 0.10
    
    # 根据任务类型调整
    if scenario_info.get('is_attacker'):
        prior['ATTACK'] += 0.15
        prior['DEFENSE'] -= 0.10
    
    # 归一化
    total = sum(prior.values())
    for k in prior:
        prior[k] /= total
    
    return prior
```

**似然函数建模**。似然函数P(行为 | 意图)通过历史数据学习得到，采用条件概率表形式存储：

```python
# 意图-行为似然矩阵示例
```

**后验概率计算**。综合先验和似然，计算各意图的后验概率：

```python
def compute_posterior(prior, likelihood, observation):
    """计算意图后验概率"""
    posterior = {}
    
    for intention in prior.keys():
        log_prob = math.log(prior[intention])
        for behavior, prob in likelihood[intention].items():
            if behavior in observation:
                # 对数相加（避免下溢）
                log_prob += math.log(prob) * observation[behavior]
        posterior[intention] = math.exp(log_prob)
    
    # 归一化
    total = sum(posterior.values())
    if total > 0:
        for k in posterior:
            posterior[k] /= total
    
    return posterior
```

### 3.5 多假设追踪框架

多假设追踪（MHT）框架支持同时维护多个可能的意图假设及其置信度，在信息不确定条件下提供更稳健的推断结果。

**假设管理机制**。系统采用假设池管理所有候选意图假设：

```python
class HypothesisPool:
    def __init__(self):
        self.hypotheses = []  # 假设列表
        self.max_hypotheses = 5  # 最大假设数量
        self.prune_threshold = 0.05  # 剪枝阈值
    
    def add_hypothesis(self, intention, probability, supporting_evidence):
        """添加新假设"""
        hypothesis = {
            'intention': intention,
            'probability': probability,
            'evidence': supporting_evidence,
            'timestamp': current_time
        }
        self.hypotheses.append(hypothesis)
        self._prune_and_normalize()
    
    def _prune_and_normalize(self):
        """剪枝并归一化"""
        # 删除低概率假设
        self.hypotheses = [
            h for h in self.hypotheses 
            if h['probability'] >= self.prune_threshold
        ]
        
        # 限制假设数量
        self.hypotheses.sort(key=lambda x: x['probability'], reverse=True)
        self.hypotheses = self.hypotheses[:self.max_hypotheses]
        
        # 归一化概率
        total = sum(h['probability'] for h in self.hypotheses)
        if total > 0:
            for h in self.hypotheses:
                h['probability'] /= total
```

**假设置信度更新**。每个决策周期，根据新观测更新所有假设的置信度：

```python
def update_hypotheses(observation):
    """更新所有假设的置信度"""
    for hypothesis in hypothesis_pool.hypotheses:
        # 计算似然比
        likelihood_ratio = compute_likelihood_ratio(
            hypothesis['intention'], 
            observation
        )
        
        # 贝叶斯更新
        prior = hypothesis['probability']
        posterior = prior * likelihood_ratio
        hypothesis['probability'] = posterior
    
    # 归一化
    hypothesis_pool._prune_and_normalize()
```

### 3.6 意图预测与时间扩展

意图预测在意图推断的基础上，增加时间维度的延伸，实现对未来态势的预判。

**时间扩展推断**。系统支持预测未来N个决策周期的意图分布：

```python
def predict_intention(current_hypotheses, n_steps):
    """预测未来n步的意图分布"""
    predictions = []
    current_distribution = {
        h['intention']: h['probability'] 
        for h in current_hypotheses
    }
    
    for step in range(n_steps):
        # 意图转移概率矩阵（根据历史数据分析得到）
        transition_matrix = {
            'ATTACK': {'ATTACK': 0.7, 'DEFENSE': 0.1, 'RECON': 0.1, 'RETREAT': 0.1},
            'DEFENSE': {'ATTACK': 0.2, 'DEFENSE': 0.6, 'RECON': 0.1, 'RETREAT': 0.1},
            'RECON': {'ATTACK': 0.3, 'DEFENSE': 0.1, 'RECON': 0.5, 'RETREAT': 0.1},
            'RETREAT': {'ATTACK': 0.1, 'DEFENSE': 0.1, 'RECON': 0.1, 'RETREAT': 0.7}
        }
        
        # 计算下一步分布
        next_distribution = {}
        for current_intent, prob in current_distribution.items():
            for next_intent, trans_prob in transition_matrix[current_intent].items():
                if next_intent not in next_distribution:
                    next_distribution[next_intent] = 0
                next_distribution[next_intent] += prob * trans_prob
        
        predictions.append(next_distribution.copy())
        current_distribution = next_distribution
    
    return predictions
```

### 3.7 置信度评估机制

置信度评估为意图推断结果提供可靠性度量，支持决策层根据置信度调整决策策略。

**信息熵度量**。采用信息熵作为推断不确定性的度量指标：

```python
def compute_confidence_entropy(posterior_distribution):
    """计算推断置信度（基于信息熵）"""
    # 信息熵公式: H = -Σ P_i * log(P_i)
    entropy = 0
    for prob in posterior_distribution.values():
        if prob > 0:
            entropy -= prob * math.log(prob)
    
    # 归一化到 [0, 1]
    max_entropy = math.log(len(posterior_distribution))
    normalized_entropy = entropy / max_entropy
    
    # 置信度 = 1 - 归一化熵
    confidence = 1 - normalized_entropy
    
    return confidence
```

**置信度等级划分**：

| 置信度区间 | 等级标识 | 决策建议 |
|-----------|---------|---------|
| 0.8-1.0 | 高置信度 | 可直接基于推断结果决策 |
| 0.5-0.8 | 中置信度 | 建议收集更多信息确认 |
| 0.2-0.5 | 低置信度 | 保持观望，保守决策 |
| 0.0-0.2 | 极低置信度 | 放弃推断，随机应变 |

```python
def check_2bop_see(self, bop_a, bop_b, mod=0):
    """综合通视、遮蔽、掩蔽、飞行单位规则的观察判定"""
    if mod:
        bop_a, bop_b = bop_b, bop_a
    
    pos_a = bop_a['cur_hex']
    pos_b = bop_b['cur_hex']
    
    # 获取观察距离约束
    if isinstance(bop_a.get('observe_distance'), dict):
        max_distance = bop_a['observe_distance'].get(bop_b['type'], 0)
    else:
        return False
    
    # 检查距离
    hex_distance = self.get_distance(pos_a, pos_b)
    if hex_distance > max_distance or max_distance == 0:
        return False
    
    # 确定观察模式
    fly_a = 1 if bop_a['type'] == 3 else 0
    fly_b = 1 if bop_b['type'] == 3 else 0
    see_mode = 1 if fly_a == 1 and fly_b == 1 else \
               2 if fly_a == 1 and fly_b == 0 else \
               3 if fly_a == 0 and fly_b == 1 else 0
    
    # 通视检查
    if not self.can_see(pos_a, pos_b, see_mode):
        return False
    
    # 应用遮蔽修正
    distance_threshold = max_distance
    if bop_b['type'] in [1, 2]:  # 地面或车辆单位
        if bop_b.get('move_state') == 4:  # 隐蔽状态
            distance_threshold //= 2
        if self.is_in_cover(pos_b):  # 掩蔽地形内
            distance_threshold //= 2
    
    return hex_distance <= distance_threshold
```

```python
def extract_behavior_features(operator_history):

    if not operator_history:

        'approach_trend': 0,  # 接近/远离趋势

    positions = [op['cur_hex'] for op in operator_history]
    if len(positions) < 2:
        return features
    
    # 计算速度特征
    velocities = []
    for i in range(1, len(positions)):
        dist = self.get_distance(positions[i], positions[i-1])
        velocities.append(dist)
    
    if velocities:
        features['avg_velocity'] = sum(velocities) / len(velocities)
        if len(velocities) > 1:
            features['velocity_variance'] = statistics.variance(velocities)
    
    # 计算方向变化
    directions = []
    for i in range(1, len(positions)):
        direction = get_direction(positions[i-1], positions[i])
        directions.append(direction)
    
    if len(directions) > 1:
        changes = sum(1 for i in range(1, len(directions)) if directions[i] != directions[i-1])
        features['direction_changes'] = changes / (len(directions) - 1)
    
    return features
```

**贝叶斯意图推断**：

```python
class BayesianIntentionClassifier:
    """贝叶斯意图分类器"""
    
    def __init__(self):
        self.priors = {
            'ATTACK': 0.25,
            'DEFENSE': 0.25,
            'RECON': 0.25,
            'RETREAT': 0.25
        }
        self.likelihoods = {}  # P(behavior|intention)
    
    def compute_posterior(self, behavior_features, observation):
        """计算意图后验概率"""
        posterior = {}
        
        for intention in self.priors.keys():
            log_prob = math.log(self.priors[intention])
            
            # 对数似然累加
            for feature, value in behavior_features.items():
                if feature in self.likelihoods.get(intention, {}):
                    likelihood = self.likelihoods[intention][feature].get(value, 0.01)
                    log_prob += math.log(likelihood)
            
            posterior[intention] = math.exp(log_prob)
        
        # 归一化
        total = sum(posterior.values())
        if total > 0:
            for k in posterior:
                posterior[k] /= total
        
        return posterior
```
