# 态势感知与意图识别算法分析及优化方案

## 1. 现有算法架构深度分析

### 1.1 系统现状概述
当前系统处于从**规则驱动（Rule-based）**向**数据驱动（Data-driven）**转型的关键阶段。现有的代码逻辑主要依赖几何计算和启发式规则来处理态势感知和决策任务，但项目已积累了高质量的结构化数据（`WG-StateGraph`, `WargameData_mini01`），为引入先进的深度学习模型奠定了坚实基础。

### 1.2 核心模块分析

#### A. 态势感知模块 (Situation Awareness)
*   **当前实现机制**:
    *   **核心代码**: `AIs/map.py`
    *   **工作流程**: 
        1.  **数据输入**: 接收仿真环境的原始观测数据（Observation）。
        2.  **几何计算**: 利用射线投射法（Ray Casting）和六角格坐标系（Hex Grid）计算算子间的通视关系（Visibility）。
        3.  **状态更新**: 实时维护战场地图对象（Map），更新地形遮蔽、高程差等环境属性。
    *   **局限性**: 仅能处理“可见性”和“距离”等物理层面的态势，缺乏对“威胁度”、“控制域”等高层语义态势的理解。
*   **数据驱动潜力**:
    *   **数据集支持**: `datasets/WG-StateGraph` 提供了完整的态势图数据。
    *   **特征维度**: `node_features` 包含108维细粒度特征（血量、弹药、机动状态、疲劳度等），非常适合构建**图神经网络（GNN）**。

#### B. 意图识别模块 (Intention Recognition)
*   **当前实现机制**:
    *   **核心代码**: `ai/agent.py`
    *   **工作流程**:
        1.  **规则判断**: 基于简单的优先级队列（Priority Queue）选择攻击目标。
        2.  **状态机**: 使用有限状态机（FSM）管理算子的“机动”、“射击”、“补给”等状态切换。
    *   **局限性**: 无法预测敌方的长时序战术意图（如“侧翼包抄”、“诱敌深入”），只能对当前帧状态做出反应。
*   **架构设计痕迹**:
    *   `AIs/utils.py` 中保留了分层架构的设计接口：**意图层 (Intention)** -> **筹划层 (Planning)** -> **执行层 (Execution)**，这与业界先进的认知架构（Cognitive Architecture）一致。
    *   **数据支持**: `datasets/WargameData_mini01` 中的 `tank_enemy` 字段记录了敌方坦克的未来7步轨迹，是训练**序列预测模型（LSTM/Transformer）**的理想标签。

## 2. 算法优化方案 (Optimization Scheme)

为实现从“规则智能”到“混合智能”的跨越，本方案提出以下四大优化维度的技术路线。

### 2.1 算法性能优化 (Performance)
针对兵棋推演高并发、低延迟的要求，重点优化推理效率。
*   **模型压缩**:
    *   **量化 (Quantization)**: 采用 Post-Training Quantization (PTQ) 技术，将 GNN 和 LSTM 模型从 FP32 压缩至 **INT8**，预计模型体积减少 75%，推理速度提升 3 倍。
    *   **剪枝 (Pruning)**: 对 Transformer 模型进行结构化剪枝，移除冗余的 Attention Head。
*   **推理加速**:
    *   **算子融合**: 使用 **TensorRT** 或 **ONNX Runtime** 部署模型，自动融合图聚合（Message Passing）中的零散算子。
    *   **批处理 (Batching)**: 在多智能体仿真中，采用 Dynamic Batching 技术合并多个 Agent 的推理请求。

### 2.2 准确性提升 (Accuracy)
通过引入深度学习技术，提升对复杂战场环境的理解能力。
*   **特征工程升级**:
    *   **异构图建模 (Heterogeneous Graph)**: 构建包含“坦克”、“步兵”、“地形”、“城市”多种节点类型的异构图，利用 **HGT (Heterogeneous Graph Transformer)** 捕捉不同实体间的语义交互。
    *   **时空特征融合**: 结合 CNN 提取地图的空间特征（Spatial）和 LSTM 提取轨迹的时间特征（Temporal），构建 **Conv-LSTM** 混合模型。
*   **集成学习 (Ensemble Learning)**:
    *   构建 **Stacking** 框架：底层使用 GNN 和规则引擎分别输出预测，顶层使用 XGBoost 进行决策融合，兼顾深度模型的泛化性和规则系统的稳定性。

### 2.3 实时性改进 (Real-time)
确保算法在毫秒级时间窗内完成态势更新与决策。
*   **增量式图计算 (Incremental Graph Computing)**:
    *   放弃每帧全量重建图的做法。仅对发生位移或状态变化的算子更新其邻域连接，将图更新复杂度从 $O(N^2)$ 降低至 $O(K)$（K为活跃节点数）。
*   **流式架构**:
    *   引入 **Kafka** + **Flink** 构建流式计算管道，实时处理战场事件流（Event Stream），实现即时的威胁预警。

### 2.4 鲁棒性增强 (Robustness)
提升算法在不确定性环境和对抗条件下的稳定性。
*   **对抗训练 (Adversarial Training)**:
    *   在训练阶段引入对抗样本（Adversarial Examples），如微小的坐标扰动或状态噪声，迫使模型学习更鲁棒的特征表示。
*   **物理规则约束 (Physics-informed Learning)**:
    *   在 Loss 函数中加入物理约束项（如：移动距离 $\le$ 最大速度 $\times$ 时间），确保深度学习模型的输出不违反兵棋推演的物理规则。
*   **异常检测**:
    *   部署基于 AutoEncoder 的异常检测模块，实时监控输入数据的分布漂移（Distribution Shift），防止模型在OOD（Out-of-Distribution）数据上失效。

## 3. 实施路线图 (Implementation Roadmap)

### 第一阶段：基准模型构建 (Baseline Construction)
*   **周期**: 第 1-4 周
*   **目标**: 跑通数据流，建立深度学习基准模型。
*   **关键任务**:
    1.  [态势感知] 基于 `WG-StateGraph` 训练 GCN/GAT 模型，实现态势评分预测。
    2.  [意图识别] 基于 `WargameData_mini01` 训练 LSTM 模型，实现坦克轨迹预测。
    3.  [系统集成] 搭建 Python 推理接口，打通仿真环境与 AI 模型的交互。
*   **交付物**: 基准模型权重文件、训练评估报告（Accuracy/F1-score）。

### 第二阶段：性能与架构优化 (Optimization)
*   **周期**: 第 5-8 周
*   **目标**: 提升推理速度，优化模型结构。
*   **关键任务**:
    1.  [性能] 完成模型的 INT8 量化与 TensorRT 部署，单帧推理延迟 < 20ms。
    2.  [架构] 升级为 HGT 异构图模型，引入物理规则约束 Loss。
    3.  [实时性] 实现增量式图更新算法。
*   **交付物**: 加速后的推理引擎、性能测试报告（FPS、Latency）。

### 第三阶段：全系统集成与验证 (Integration & Validation)
*   **周期**: 第 9-12 周
*   **目标**: 实战部署，验证综合效能。
*   **关键任务**:
    1.  [集成] 将 AI 模块完整嵌入兵棋推演平台，替换原有的规则模块。
    2.  [对抗] 引入对抗训练，进行大规模自我对弈（Self-Play）以提升策略水平。
    3.  [测试] 组织人机对抗测试，评估胜率与战术合理性。
*   **交付物**: 完整优化的 AI 系统、人机对抗分析报告。
